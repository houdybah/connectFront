Guide d'utilisation des signatures

Gestion des périodes de validité

Les signatures peuvent être configurées avec des périodes de validité spécifiques :

  • Date de début : Date à partir de laquelle la signature sera utilisée
  • Date de fin : Date jusqu'à laquelle la signature sera utilisée
  • Si vous laissez ces champs vides, la signature sera valide indéfiniment
Signatures par bureau
  • Signature spécifique : valide uniquement pour un bureau
  • Signature générique (ALL) : valable pour tous les bureaux
Ordre de priorité (aligné avec le contrôleur)

Le système recherche les signatures dans cet ordre :

  1. Signature actuelle : Signature spécifique au bureau ET actuellement valide
  2. Dernière signature valide : Plus récente signature valide pour le bureau
  3. Signature avec priorité : Priorité aux signatures sans date de fin
  4. Signature générique (ALL) : Si aucune signature spécifique
Types de test disponibles
  • Dernière valide : Trouve la dernière signature valide
  • Actuelle : Signature actuellement active
  • Avec priorité : Priorité aux signatures permanentes
Nouvelles fonctionnalités
  • Validation : Vérifier si une signature est valide à une date
  • Filtres avancés : Filtrer par bureau, date, validité
  • Statuts enrichis : Active, Expirée, En attente, etc.
  • Alertes d'expiration : Signatures expirant bientôt
  • Export CSV : Exporter les données des signatures
  • Tri intelligent : Trier par différents critères
Conseils d'utilisation
  • Testez toujours vos signatures après les avoir créées
  • Rafraîchissez le cache après modifications importantes
  • Surveillez les dates d'expiration
  • Utilisez les filtres pour organiser vos signatures
  • Exportez régulièrement pour backup
